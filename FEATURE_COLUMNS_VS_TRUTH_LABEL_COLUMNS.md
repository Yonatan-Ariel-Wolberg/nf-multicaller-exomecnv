# Feature Columns vs Truth-Label Columns

This document clarifies which columns are used as **model features** and which belong to the **truth-label TSV** for XGBoost training (`bin/train_xgboost.py`).

## 1) Input feature matrix (`*_features.tsv`)

Generated by `bin/feature_extraction.py`.

### Identifier / join columns (present in features TSV)
- `sample_id`
- `chrom`
- `start`
- `end`
- `cnv_type`  
  - Encoded in features as numeric: `1` for DUP, `0` for DEL/other.

### Feature columns used by the model
The model uses numeric feature columns after removing identifier columns.
Typical feature columns include:

- Structural/location: `chrom_encoded`, `cnv_size`, `size_label`
- Concordance: `concordance`
- Per-caller support flags: `is_<caller>` (e.g., `is_canoes`, `is_clamms`, ...)
- Per-caller normalized quality: `qual_norm_<caller>`
- Aggregate quality: `max_qual_norm`, `mean_qual_norm_supported`
- Genomic/probe/depth: `n_probes`, `n_probes_flank`, `rd_ratio`, `gc_content`, `mappability`
- L2R stats: `l2r_mean`, `l2r_dev`, `l2r_flank_diff`
- Caller-specific secondary metrics (when available): `xhmm_rd`, `cnvkit_weight`, `cnvkit_log2`, `dragen_sm`, `dragen_sd`
- INDELIBLE split-read metrics (when available): `total_sr`, `sr_entropy`, `mapq_avg`, `dual_split`

> Note: Exact per-caller columns depend on which caller VCFs are provided, but the expected pattern is `is_<caller>` and `qual_norm_<caller>`.

## 2) Truth-label TSV (`--truth_labels`)

Required minimum columns:
- `sample_id`
- `chrom`
- `start`
- `end`
- `cnv_type` (e.g., `DEL` / `DUP`)
- `truth_label`

### Meaning
- `truth_label = 1`: true CNV
- `truth_label = 0`: false positive

## 3) How features and labels are matched

`train_xgboost.py` performs an **inner join** between features and truth labels on:
- `sample_id`
- `chrom`
- `start`
- `end`
- `cnv_type`

Only rows that match on all five keys are used for training.

## 4) What becomes `X` vs `y`

After joining:
- `y` (target) = `truth_label`
- `X` (model input) = numeric columns from merged data after dropping identifier columns:
  - dropped identifiers: `sample_id`, `chrom`, `cnv_type`

So:
- **Truth-label column:** `truth_label`
- **Feature columns:** numeric columns retained after dropping `sample_id`, `chrom`, and `cnv_type`.
  - In the current implementation, numeric coordinate columns like `start` and `end` are retained and therefore can be part of `X`.
